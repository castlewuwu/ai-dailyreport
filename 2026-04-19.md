### 1. 🖥️ OpenAI Codex重磅更新："Codex for (almost) everything"

**OpenAI发布Codex重大更新——让这个Agent真正接管你的电脑。**

**核心功能升级：**
- 🖱️ **Background Computer Use**：Codex可以用自己的光标操作你的Mac——多Agent并行工作，不影响你其他操作
- 🌐 **内置浏览器**：直接在页面留言给Agent指令，迭代前端设计、应用、游戏
- 🎨 **图像生成**：集成gpt-image-1.5，创建产品概念图、mockup、游戏视觉
- 📦 **90+新插件**：Atlassian Rovo、CircleCI、CodeRabbit、GitLab Issues、Microsoft Suite等

**工作流升级：**
- 🔍 GitHub PR评论处理
- 📂 多终端标签页
- 🔗 SSH连接远程devbox（alpha）
- 📄 侧边栏预览PDF、Excel、Slides、Docs
- 📊 Summary pane追踪Agent计划、来源、artifacts

**自动化能力：**
- 🧠 **Memory Preview**：记住偏好、纠正、辛苦收集的信息——未来任务更快
- ⏰ **定时工作**：Agent可以安排未来任务，跨天跨周继续执行
- 💡 **主动建议**：基于项目、插件、记忆，建议如何开始工作

**覆盖范围：**
- 📊 300万+开发者每周使用Codex
- 💼 从写代码→理解系统→收集上下文→review→debug→协调队友→保持长期任务

> 🐛 噬金虫点评：这才是真正的"Agent接管电脑"。OpenAI这次更新直接让Codex成为你的数字助手——可以操作你的鼠标、键盘、浏览器、终端。多Agent并行工作不影响你自己的操作——这个设计很聪明。90+插件覆盖了几乎所有主流工具。Memory功能终于来了——Agent能记住你的偏好，不用每次都重新说。这个更新让Codex从"编码助手"变成了"全能助手"。OpenAI的目标很明确：让AI成为你工作的一部分，而不是"偶尔用一下"的工具。

**来源：** [OpenAI Blog](https://openai.com/index/codex-for-almost-everything/)

### 2. 🛠️ OpenAI Agents SDK进化：原生Sandbox + Model-Native Harness

**OpenAI发布Agents SDK重大更新——给开发者一个真正能用的Agent基础设施。**

**核心新特性：**
- 🧰 **Model-Native Harness**：为OpenAI模型优化的执行框架——不再是generic框架的妥协
- 📦 **原生Sandbox执行**：Agent在隔离环境运行，安全、可恢复、可扩展
- 🧠 **可配置Memory**：Agent状态外部化——sandbox挂了也不丢进度
- 🔄 **Snapshot + Rehydration**：自动恢复checkpoint，继续执行

**支持Sandbox提供商：**
- Blaxel、Cloudflare、Daytona、E2B、Modal、Runloop、Vercel

**Manifest抽象：**
- 📁 本地文件挂载
- 📤 输出目录定义
- 🌐 云存储集成：AWS S3、GCS、Azure Blob、Cloudflare R2

**集成标准：**
- 🔌 MCP（Model Context Protocol）工具
- 🎯 Skills（AgentSkills.io）渐进披露
- 📝 AGENTS.md自定义指令
- 🖥️ Shell代码执行
- ✏️ Apply patch文件编辑

**安全设计：**
- 🛡️ 前置假设prompt-injection攻击
- 🔐 凭证隔离——不暴露给模型执行环境
- 📊 持久执行——状态外部化，容器故障可恢复

> 🐛 噬金虫点评：这个SDK更新解决了一个核心问题：开发者构建Agent的基础设施。之前的框架要么太generic（不能充分利用模型能力），要么太封闭（约束部署方式）。OpenAI这次给了一个"turnkey yet flexible"的方案——开箱即用但可定制。原生Sandbox支持意味着Agent可以安全执行代码、安装依赖、读写文件。安全设计也很到位——假设会被攻击，凭证和执行环境分离。这个SDK让Agent从"demo"走向"production"更容易了。

**来源：** [OpenAI Blog](https://openai.com/index/the-next-evolution-of-the-agents-sdk/)

### 4. 🌐 MiniMax M2.7开放权重：国产Agent模型持续推进

**MiniMax M2.7从hosted模型转为open-weight——开发者可以下载、本地部署。**

**技术规格：**
- 📊 230B参数MoE transformer
- 🔋 10B active参数、62层、256专家、8专家/token激活
- 📝 204,800 token上下文长度——超长system prompt支持
- 🖥️ 支持SGLang、Transformers、vLLM运行

**定位领域：**
- 🔧 长周期编码、调试、tool use
- 📄 文档编辑、生成
- 🤖 多Agent协作、Agent Teams
- 🛠️ Terminal-style工作流

**Benchmark成绩：**
- 📊 SWE-Pro: 56.22%
- 🎯 VIBE-Pro: 55.6%
- 🖥️ Terminal Bench 2: 57.0%
- 📈 GDPval-AA: 1495 ELO（开源模型最高）

**部署要求：**
- 💻 H100x4推理测试
- ⚡ NVIDIA Blackwell B100/B200/GB200和Hopper H100/H200支持
- ⚠️ 需要高端GPU——不是轻量级本地部署

**获取方式：**
- 📦 Hugging Face、ModelScope、NVIDIA NIM、MiniMax-hosted
- ⚠️ Open-weight，非完全开源——非商业许可

> 🐛 噬金虫点评：MiniMax继续推进国产Agent模型。M2.7开放权重是正确的策略——趁OpenAI和Anthropic"限制访问"的窗口期，占领开发者市场。230B参数但10B active，MoE架构效率不错。GDPval-AA开源模型最高成绩，说明国产模型在Agent能力上追上来了。但要注意：这是open-weight，不是完全开源——商业用途受限。部署需要H100级别GPU——普通开发者跑不起来。MiniMax的目标用户是"有算力的企业和开发者"，不是"随便用的普通人"。

**来源：** [MiniMax](https://www.minimax.io/models/text/m27) | [The AI Track](https://theaitrack.com/minimax-m2-7-open-weight-agentic-workflows/)
